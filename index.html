How to use

  1. Plug in the SL MkIII and press InControl on the unit.
  2. Click Connect MIDI and choose the InControl port.
  3. Click any control, pick a colour — with Live send on it lights up instantly.
  4. Save your layout as .json, or export a .syx to replay anywhere.

Needs Chrome, Edge, or Opera (Web MIDI). Colours are 7-bit per channel on the hardware, so the on-screen colour is approximate.

Mappings export to the template · colours flow to Live Colours & Bridge

Template + colour editor

Create or load a Components template and edit every control's MIDI mapping (type, channel, CC/Note, min/max, behaviour, velocity). Pads, faders and the 8×2 buttons also take a colour that's stored in Live Colours and pushed live by the Bridge — the mappings live in the .syx, the colours live in InControl. The template codec is bit-exact and recomputes the file checksum, so exports import cleanly into Components.

InControl Studio. To drive the hardware: put the SL MkIII in InControl mode, click Connect MIDI (top-right), then in the Live engine bar set SL in/SL out to the “InControl” port and Destination to a virtual port your DAW/synth listens to (IAC / loopMIDI), and press Start engine. Then the Grid button toggles the pads between playable and the step sequencer; transport buttons drive playback; the up/down buttons by the pads page patterns. Import a Components .syx to start from a template.
Live engine:
The Bridge holds your Live Colours layout on the unit and remaps its InControl messages to a destination MIDI port — colours + custom mappings at once. A browser can't create a virtual port, so pick an existing one (an IAC bus on macOS, loopMIDI on Windows). For a headless service that makes its own port, use the Node bridge in /bridge.
